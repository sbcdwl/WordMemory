# 开发指南

本文档面向希望修改或扩展 WordMemory 的开发者。

## 代码架构

### 三层架构

```text
┌─────────────────────────────────────────────────────────────────┐
│                        用户界面层 (Views)                       │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐          │
│  │ main_window  │ │ import_dialog│ │settings_dialog│          │
│  └──────────────┘ └──────────────┘ └──────────────┘          │
│  ┌──────────────┐                                            │
│  │ stats_window │                                            │
│  └──────────────┘                                            │
├─────────────────────────────────────────────────────────────────┤
│                       业务逻辑层 (Controllers)                  │
│  ┌──────────────┐ ┌──────────────┐                           │
│  │   importer   │ │  scheduler   │                           │
│  └──────────────┘ └──────────────┘                           │
├─────────────────────────────────────────────────────────────────┤
│                         数据层 (Models)                         │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐          │
│  │  database    │ │    book      │ │    word      │          │
│  └──────────────┘ └──────────────┘ └──────────────┘          │
│  ┌──────────────┐                                            │
│  │review_record │                                            │
│  └──────────────┘                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 模块职责

| 模块                 | 职责                      | 依赖                           |
| :------------------- | :------------------------ | :----------------------------- |
| `database.py`        | 数据库连接管理、线程安全  | SQLite3                        |
| `book.py`            | 词书 CRUD 操作            | database                       |
| `word.py`            | 单词 CRUD 操作            | database                       |
| `review_record.py`   | 复习记录 CRUD + SM-2 算法 | database                       |
| `importer.py`        | JSON 解析和数据导入       | book, word, review_record      |
| `scheduler.py`       | 每日学习队列生成          | book, word, review_record      |
| `main_window.py`     | 主界面和学习交互          | scheduler, word, review_record |
| `import_dialog.py`   | 导入文件对话框            | importer                       |
| `settings_dialog.py` | 设置界面                  | database                       |
| `stats_window.py`    | 统计信息展示              | database, scheduler            |

## 开发规范

### 命名规范

| 类型     | 规范       | 示例                              |
| :------- | :--------- | :-------------------------------- |
| 类名     | PascalCase | `MainWindow`, `ReviewRecord`      |
| 方法名   | snake_case | `get_by_id()`, `_load_settings()` |
| 私有方法 | 下划线前缀 | `_get_connection()`               |
| 常量     | 大写蛇形   | `MAX_RETRY_COUNT`                 |

### 代码风格

- 使用 4 空格缩进
- 每行不超过 120 字符
- 类和方法添加 docstring 注释
- 导入顺序：标准库 → 第三方库 → 本地模块

### Git 提交规范

```text

style: 代码格式调整

```

示例：

## 如何添加新功能

### 示例1：添加新的学习模式

**步骤1**：在 `settings` 表中添加模式选项

```python
# database.py - _init_default_settings()
default_settings = {
    # ... 现有设置
    'learning_style': 'sequential',  # sequential / shuffle / test
}
```

**步骤2**：在设置对话框添加控件

```python
# settings_dialog.py
self.combo_style = QComboBox()
self.combo_style.addItem("顺序学习", "sequential")
self.combo_style.addItem("随机打乱", "shuffle")
self.combo_style.addItem("测试模式", "test")
```

**步骤3**：在调度器中实现逻辑

```python
# scheduler.py
def _apply_learning_style(self, queue):
    style = db.get_setting('learning_style', 'sequential')
    if style == 'shuffle':
        random.shuffle(queue)
    elif style == 'test':
        # 只显示复习词
        queue = [q for q in queue if q['type'] == 'review']
    return queue
```

**步骤4**：在主窗口调用

```python
# main_window.py
queue = self.scheduler.get_combined_queue()
queue = self.scheduler._apply_learning_style(queue)
```

### 示例2：添加音频播放功能

**步骤1**：在 `utils/` 中创建音频工具

```python
# src/utils/audio.py
import pygame

def play_audio(file_path):
    pygame.mixer.init()
    pygame.mixer.music.load(file_path)
    pygame.mixer.music.play()
```

**步骤2**：在主窗口添加播放按钮

```python
# main_window.py
self.btn_audio = QPushButton("🔊 发音")
self.btn_audio.clicked.connect(self._on_play_audio)

def _on_play_audio(self):
    if self.current_word and self.current_word.audio_local_path:
        play_audio(self.current_word.audio_local_path)
```

**步骤3**：在设置中添加开关

```python
# settings_dialog.py
self.check_audio = QCheckBox("自动播放发音")
```

### 示例3：添加导入进度条

**步骤1**：修改 `ImportWorker` 发送进度信号

```python
# import_dialog.py
class ImportWorker(QThread):
    progress = pyqtSignal(int, str)
    
    def run(self):
        total = len(data_list)
        for i, data in enumerate(data_list):
            # 处理每个单词
            self.progress.emit(int((i+1)/total*100), f"导入 {i+1}/{total}")
```

**步骤2**：连接信号到进度条

```python
self.worker.progress.connect(self._update_progress)

def _update_progress(self, value, text):
    self.progress_bar.setValue(value)
    self.status_label.setText(text)
```

## 常见问题

### Q: 数据库连接报错 "SQLite objects created in a thread can only be used in that same thread"

**A**: 确保每个线程使用独立的数据库连接。`database.py` 已使用 `threading.local` 实现。

### Q: 如何重置所有数据？

**A**: 删除 `data/word_memory.db` 文件，重新启动程序。

### Q: 修改代码后界面没有变化？

**A**: 检查是否保存了文件，并重新运行 `python main.py`。

### Q: 导入 JSON 时提示 "JSON 解析失败"？

**A**: 使用 JSON 验证工具检查文件格式，确保 UTF-8 编码。
